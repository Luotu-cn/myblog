# assert(断言)

# assert.AssertionError 类

表明断言的失败。 assert 模块抛出的所有错误都将是 AssertionError 类的实例。

new assert.AssertionError(options) 简要说明：使用options参数设置实例的相关属性

const { message } = new assert.AssertionError({
  actual: 1,
  expected: 2,
  operator: 'strictEqual'
});

# 🔊上例中使用了ES6语法--解构，参见解构赋值-菜鸟

使用场景：当需要自定义AssertionError 类的时候

详见：传送门

# assert

assert(value[, message])

简要说明：检查value参数是否为真

# 🔊其中参数[,message]表示选输项

使用场景：

# assert.strictEqual

assert.strictEqual(actual, expected[, message])

简要说明：测试 actual 参数和 expected 参数之间的严格相等性，使用 SameValue比较。

const assert = require('assert').strict;

// 1 !== 2
assert.strictEqual(1, 2);

// OK
assert.strictEqual(1, 1);

//不等
assert.strictEqual('Hello foobar', 'Hello World!');

// AssertionError [ERR_ASSERTION]: apples 1 !== oranges 2
const apples = 1;
const oranges = 2;
assert.strictEqual(apples, oranges, `apples ${apples} !== oranges ${oranges}`);

// TypeError: Inputs are not identical
assert.strictEqual(1, '1', new TypeError('Inputs are not identical'));

如果值不严格相等，则抛出 AssertionError，并将 message 属性设置为等于 message 参数的值。 如果未定义 message 参数，则会分配默认错误消息。 如果 message 参数是 Error 的实例，则它将被抛出而不是 AssertionError。

详见：传送门

# assert.deepStrictEqual

assert.deepStrictEqual(actual, expected[, message])

简要说明：测试 actual 参数和 expected 参数之间的深度相等

const assert = require('assert').strict;

// 1.失败，因为 1 !== '1'。
assert.deepStrictEqual({ a: 1 }, { a: '1' });

// 2.以下对象没有自身属性。
const date = new Date();
const object = {};
const fakeDate = {};
Object.setPrototypeOf(fakeDate, Date.prototype);
：

assert.deepStrictEqual(object, fakeDate);
// 原型不同
assert.deepStrictEqual(date, fakeDate);
// 类型标签不同：

// 3.通过，因为使用 SameValue 比较。
assert.deepStrictEqual(NaN, NaN);

// 4.解封装后的数字不同：
assert.deepStrictEqual(new Number(1), new Number(2));

// 5.通过，因为对象与解封装后的字符串都是相同的。
assert.deepStrictEqual(new String('foo'), Object('foo'));

// 6.通过。
assert.deepStrictEqual(-0, -0);

// 7.使用 SameValue 比较的零不同：
assert.deepStrictEqual(0, -0);

// 8.
const symbol1 = Symbol();
const symbol2 = Symbol();
assert.deepStrictEqual({ [symbol1]: 1 }, { [symbol1]: 1 });

// 通过，因为在两个对象上的 symbol 相同。
assert.deepStrictEqual({ [symbol1]: 1 }, { [symbol2]: 1 });

// 9.
const weakMap1 = new WeakMap();
const weakMap2 = new WeakMap([[{}, {}]]);
const weakMap3 = new WeakMap();
weakMap3.unequal = true;

// 通过，因为无法比较条目。
assert.deepStrictEqual(weakMap1, weakMap2);

// 失败，因为 weakMap3 有一个 weakMap1 不包含的属性：
assert.deepStrictEqual(weakMap1, weakMap3);

如果值不相等，则抛出 AssertionError，并将 message 属性设置为等于 message 参数的值。 如果未定义 message 参数，则会分配默认错误消息。 如果 message 参数是 Error 的实例，那么它将被抛出而不是 AssertionError。

# assert.notStrictEqual

assert.notStrictEqual(actual, expected[, message])

简要说明：测试 actual 参数和 expected 参数之间的严格不相等，使用 SameValue比较

const assert = require('assert').strict;

// 通过
assert.notStrictEqual(1, 2);

//// AssertionError [ERR_ASSERTION]
assert.notStrictEqual(1, 1);

// 通过
assert.notStrictEqual(1, '1');

如果值严格相等，则抛出 AssertionError，并将 message 属性设置为等于 message 参数的值。 如果未定义 message 参数，则会分配默认错误消息。 如果 message 参数是 Error 的实例，则它将被抛出而不是 AssertionError

使用场景：

详见：传送门

# assert.notDeepStrictEqual

assert.notDeepStrictEqual(actual, expected[, message])

简要说明：测试深度严格的不平等。 与 assert.deepStrictEqual() 相反。

const assert = require('assert').strict;

// 通过。
assert.notDeepStrictEqual({ a: 1 }, { a: '1' });

如果值深度且严格相等，则抛出 AssertionError，并将 message 属性设置为等于 message 参数的值。 如果未定义 message 参数，则会分配默认错误消息。 如果 message 参数是 Error 的实例，则它将被抛出而不是 AssertionError。
